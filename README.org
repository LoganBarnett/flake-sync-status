#+author:    Logan Barnett

#+language:  en

A CLI tool that queries a Nix flake for all of its ~nixosConfigurations~ and
~darwinConfigurations~, then checks each host to see whether the currently-active
generation matches what the flake would deploy.

Useful for rolling out changes piecemeal across a fleet and confirming each
host has applied the latest config.

* Usage

#+begin_src shell :exports code
flake-sync-status [OPTIONS] [FLAKE]
#+end_src

~FLAKE~ defaults to the current directory.

** Options

| Flag              | Description                                    |
|-------------------+------------------------------------------------|
| ~-j~, ~--json~    | Emit JSON instead of the table                 |
| ~--no-color~      | Suppress all ANSI color codes                  |
| ~--log-level~     | Log level (default: ~warn~)                    |
| ~-h~, ~--help~    | Print help                                     |

Color output is also suppressed when the ~NO_COLOR~ environment variable is set
([[https://no-color.org/]]).

** Human-readable output

Each row shows:

- *Hostname* - colored consistently by hash
- *System* - platform double (~x86_64-linux~, ~aarch64-darwin~, …) with each
  segment colored independently
- *Expected* - abbreviated store hash from ~nix eval~, rainbow-colored
- *Current* - abbreviated store hash from ~/run/current-system~ on the host,
  rainbow-colored; ~unreachable~ on SSH failure; ~offline~ when marked offline;
  ~eval error~ when the flake could not be evaluated
- *Sync* - green ~✓~, red ~✗~, yellow ~?~ (unreachable/eval error), or ~—~
  (offline by config)

A summary line is printed after the table, followed by per-host error details
when any host had failures.

* Offline Hosts

The tool ships NixOS and nix-darwin modules that expose a ~flakeStatus.online~
boolean option.  Import the module into a host's configuration and set the
option to ~false~ to tell the tool that the host is intentionally offline.  The
option has no effect on the host's own activation — it is only read by the CLI.

When ~flakeStatus.online = false~, the tool skips all SSH connectivity checks
and shows ~offline~ in the CURRENT column and ~—~ in the SYNC column.  Hosts
that do not import the module default to ~online = true~ and are checked
normally.

** NixOS

#+begin_src nix :exports code
{
  inputs.flake-sync-status.url = "github:LoganBarnett/flake-sync-status";

  outputs = { nixpkgs, flake-sync-status, ... }: {
    nixosConfigurations.myhost = nixpkgs.lib.nixosSystem {
      modules = [
        flake-sync-status.nixosModules.default
        {
          flakeStatus.online = false;  # mark this host as intentionally offline
        }
      ];
    };
  };
}
#+end_src

** nix-darwin

#+begin_src nix :exports code
{
  inputs.flake-sync-status.url = "github:LoganBarnett/flake-sync-status";

  outputs = { nix-darwin, flake-sync-status, ... }: {
    darwinConfigurations.myhost = nix-darwin.lib.darwinSystem {
      modules = [
        flake-sync-status.darwinModules.default
        {
          flakeStatus.online = false;  # mark this host as intentionally offline
        }
      ];
    };
  };
}
#+end_src

* How it works

For each host the tool:

1. Runs ~nix eval --raw '.#<nixos|darwin>Configurations.HOST.config.system.build.toplevel.outPath'~
   to get the store path the flake expects.
2. Connects to the host (or reads locally if it matches ~hostname~) and runs
   ~readlink -f /run/current-system~ to get the active generation's store path.
3. Compares the two paths and reports the result.

The expected path is computed from the derivation graph without building
anything; it is the deterministic output path Nix would use if the
configuration were built right now.

* Output streams

Program output (the table or JSON) goes to *stdout*.  All log output goes to
*stderr*.  This lets callers separate the two streams, e.g. pipe JSON while
watching logs:

#+begin_src shell :exports code
flake-sync-status --json . 2>/dev/null | jq '.[] | select(.in_sync | not)'
#+end_src
