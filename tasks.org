* Offline Mode: NixOS and Darwin Modules

** TODO [#A] Create nix/modules/nixos.nix
NixOS module for the =flakeStatus.online= option.  The =config= section
is intentionally empty — the option is consumed externally by the CLI,
not by the host's own activation:

#+begin_src nix
{ lib, ... }:
{
  options.flakeStatus = {
    online = lib.mkOption {
      type = lib.types.bool;
      default = true;
      description = lib.mdDoc ''
        Whether this host is reachable for flake-sync-status checks.
        When `false`, the CLI skips all connectivity checks and

        hosts that are intentionally powered off or air-gapped.
      '';
    };
  };
}
#+end_src

** TODO [#B] Query flakeStatus.online in flake.rs / query_host()
Before running =get_current_system()=, eval:
  =<configType>.<host>.config.flakeStatus.online=

If the result is the string ="false"= (=nix eval --raw= normalises bools
to strings), skip connectivity, set =current_path = None=, and push an
informational entry into =errors= such as ="host is marked offline"=.

If the attribute does not exist (eval error), assume =true=; not all
configurations will import the module.

** TODO [#B] Distinguish offline vs. unreachable in HostStatus
Currently both "host is offline" and "SSH timed out" would appear as
errors with =in_sync = None= and no =current_path=.  Add an
=offline: bool= field (or an =OfflineMarked= error variant) so
=output.rs= can render them with visually distinct text:
- Offline (by config): dim "offline" in the CURRENT column, "—" in SYNC
- Unreachable (by failure): existing red-background "unreachable"

** TODO [#B] Document the modules in README.org

* Error Handling

** TODO [#B] Replace thread expect() with graceful degradation
=crates/cli/src/flake.rs:74= — =h.join().expect("host query thread panicked")=

A panicking thread currently brings down the entire process.  Recover
the panic payload via =JoinHandle::join()= returning =Err= and convert
it to a =HostStatus= with a descriptive error entry, letting all other
hosts still render.

Example approach:

#+begin_src rust
let status = h.join().unwrap_or_else(|_| HostStatus {
  hostname: "<unknown>".to_string(),
  errors: vec!["host query thread panicked".to_string()],
  ..Default::default()
});
#+end_src

Note: if the hostname was moved into the thread, the outer closure
needs to retain a copy to populate the error row.  Track hostnames
alongside handles in a =(String, JoinHandle<_>)= pair.

** TODO [#C] Audit error variant context richness

In particular: =HostError::Ssh= already includes host and reason;

* Parallelism

** TODO [#C] Add concurrency-limit flag (--jobs / -j)
With many hosts, all queries fire simultaneously.  Add =--jobs <N>=
(default: unlimited) to cap in-flight threads.  Use a counting
semaphore (=std::sync::Semaphore=, or a channel-based token bucket) to
release slots as threads finish.

This is useful when =nix eval= is memory-intensive or when SSH to a
jump host rate-limits.

* UX Improvements

** TODO [#B] Non-zero exit code when out of sync
Useful for CI pipelines and shell =if=-blocks.  Proposed codes:
- 0: all queried hosts in sync (or all offline)
- 1: one or more hosts out of sync
- 2: one or more hosts returned errors (unreachable, eval failure)

These are additive: if some hosts are out of sync *and* some errored,
return 2 (the higher severity).
